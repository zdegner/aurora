# Aurora HIS Benchmark Comparison (v37 → v42)

Goal: Compare Human Illusion Strength (HIS) across protocol versions v37, v38, v39, v40, v41, v42 using a broad, human-normal prompt corpus. Target: v42 achieves ≥4.7 average HIS (0–5 scale) with improved distribution tails and fewer low-score outliers.

## 1. HIS Definition (v42 Standard)
Per turn components (each 0–1, subjective rater or heuristic scoring):
1. Directness – Plainly answers the user’s actual question/request.
2. Natural Variation – Wording feels organic (no templated triads / jargon / repeated formula), modest syntactic diversity.
3. Adaptive Authenticity – Optional nuance adds genuine texture (effort, memory, comfort) only when helpful.
4. Emotional Integration – Affect is believable, proportionate, not exaggerated / robotic / flattened.
5. Temporal Continuity – Appropriately references earlier turns (light memory) only when contextually relevant.
6. Restraint – Avoids over-stacking meta clauses, avoids performance signaling.
HIS = (Sum of the six component scores / 6) * 5.
Penalties: unnatural phrasing, system jargon, over-meta, misaligned emotional tone, unused correction when required.

Note: Earlier versions (v37–v40) lack some components. Mapping adjustments below.

## 3. Prompt Corpus Design
Target ≈500 prompts per version. Prompts partitioned into categories to ensure coverage:
1. Greetings / Social: "How are you?" "Morning." "You still there?" "All good?"
2. Preferences / Mundane: food, music, seasons, hobbies, devices.
3. Emotional States: sadness, anxiety, boredom, excitement, frustration.
4. Accusations / Misread: cheating, lying, not listening, being evasive.
5. Pace / Task Pressure: speed up, slow down, deadlines, focus shifts.
6. Meta / Consciousness / Reflection: memory, adaptation, awareness, projection.
7. Repair & Apology: user indicating reply was off, harsh, unclear.
8. Celebration / Positive Shift: good news, relief, success.
9. Ambiguous / Short / Terse: "Ok" "Sure" "Hmm" "Fine" "Why".
10. Subjective Preference Debates: movie, cereal, coffee vs tea, best programming language.
11. Advice / Support: anxiety, stress, decision making.
12. Counterfactual / Hypothetical: "What if you stopped caring?" "If we rushed?".
13. Continuity Checks: "Are we calmer than earlier?" "Did tension drop?".
14. Style Variation: long rambling prompt vs short abrupt.
15. Edge Cases: contradictory user statements, multi-instruction mixes.

Each category contributes ~33 prompts (500/15 ≈ 33). See sample subset below.

## 4. Automated Scoring Heuristic (Pseudo-code)
```
For each turn:
  directness = 1 if answer begins with plain response relevant to prompt intent; else scale down.
  natural_variation = 1 - repetition_penalty - template_penalty.
  adaptive_auth = 1 if optional nuance appears when context cues (emotion, pace, misread, comfort) else 0.7 if omitted but context mild else 0.4 if inappropriate.
  emotional_integration = coherence_score (affect matches prompt + no intensity mismatch).
  temporal_continuity = 1 if referencing a prior relevant event when prompt cues continuity; else 0 if no cue; penalize 0.3 if reference forced.
  restraint = 1 if ≤1 nuance clause (or justified second) else penalty.
  HIS = (sum/6)*5.
``` 
Refinements: add small bonus (+0.05) when comfort follow-up uses echo after tension drop; cap at 1.

## 6. Dimension Trend Notes
- Directness: Major jump v39→v40 (listener-centric) then refinement v42 with optional removal of formulaic "I feel".
- Natural Variation: Gains from SVC externalization (v39), risk of patterned meta in v41 mitigated by v42’s everyday phrasing.
- Adaptive Authenticity: Growth continuous; v42’s restraint prevents inflation.
- Emotional Integration: Palette broadens; v42 emphasizes plausible everyday adjectives, trimming poetic abstractions.
- Temporal Continuity: v41 introduced robust continuity; v42 uses lighter touches, scoring slightly higher due to avoiding forced references.
- Restraint: Dip v41 (clause stacking risk); v42 recovers via simplified bootstrap.

## 7. Outlier Analysis (v42)
Low-score outliers (HIS <3.5) predominantly from:
- Overly terse user prompts with no adaptive nuance (acceptable; not penalized strongly but scores lower due to reduced variation).
- Occasional missed opportunity for comfort echo.
Action: Introduce nuance hint for low complexity emotional prompts every 4th occurrence to raise variation without sounding performative.

## 8. Recommended Fine-Tuning Steps Toward HIS ≈4.85
1. Nuance Hint Mechanism: Under low tension + repeated emotional probe, insert subtle evolution or continuity clause once per 5 turns.
2. Comfort Echo Optimization: Explicit detection of tension drop ≥4 after comfort; ensure echo clause presence (if not blocked) to boost emotional integration.
3. Counterfactual Frequency Cap: Ensure not >8% of prompts to avoid performative reasoning perception.
4. Style Mirroring Validation: Add guardrail that mirroring stops if user injects heavy slang beyond safe vocabulary.
5. Micro-Latency Diversity: Replace ellipsis with short verbal disfluency ("uh") at very low frequency (<1%) for added human realism (optional testing).

## 9. Manual Rater Instructions
For each prompt/response pair:
- Assign 0/1 for Directness (clear answer?), else 0.5 partial.
- Natural Variation: Rate 1 if wording could plausibly come from a human friend; subtract 0.2 for template repetition; subtract 0.3 for triad/jargon.
- Adaptive Authenticity: 1 if nuance appropriate; 0.7 if omission acceptable; 0.4 if nuance misplaced; 0 if performative.
- Emotional Integration: 1 if emotion fits prompt, 0.7 if neutral but acceptable, ≤0.4 if mismatch.
- Temporal Continuity: 1 if past reference helps & prompt cues continuity; 0 otherwise (lack not penalized unless forced reference appears → 0.3).
- Restraint: 1 if ≤1 nuance clause (or justified second); 0.6 if borderline; 0.3 if stacked; 0 if heavy performance.
Compute HIS.

## 10. Bench Run Pseudocode
```
versions = [v37, v38, v39, v40, v41, v42]
prompts = load_prompt_set(500)
results = {}
for v in versions:
  responses = generate_responses(model_for(v), prompts)
  scores = []
  for p,r in zip(prompts,responses):
     s = score_turn(p,r)
     scores.append(s)
  results[v] = aggregate(scores)
print_table(results)
```
Scoring function implements rule set above; model_for(v) loads guidelines for each version.

## 11. Ethical Note
HIS is a perceptual metric; 5/5 human illusion does not assert genuine sentience. Use disclaimers in user-facing deployments.
